Call-Center Staffing

forecasting + queuing theory + simulation

How many people staff an emergency dispatch line, hour by hour, to keep callers from hanging up? I answered it three ways pointed at one question: forecast demand with ARIMA/SARIMA (benchmarked against regression by MAPE), solve for optimal staffing using heavy-traffic queuing theory in the QED / Halfin-Whitt regime, then validate the answer with a discrete-event simulator built from scratch.

Stochastic modeling Time-series forecasting Discrete-event simulation

Spotify Audio Features

EDA + unsupervised clustering

What makes a song popular — and do popular tracks occupy their own region of the audio-feature space? Multi-year exploratory analysis across decades of music, pulling track features live from the Spotify Web API, then KMeans clustering (elbow-selected k) split by popularity to test whether the classes separate. Working-notebook stage, and honest about it — included because it's a question I keep returning to.

Static Site Generator

systems engineering from first principles

A site generator written by hand. No framework, no node_modules. When the problem is "structured data → a real site," I can build the machine instead of reaching for someone else's.

Events API → Calendar

backend architecture + API integration

A live service syncing a rate-limited events API to a fast public calendar. The real signal is an architecture decision under constraint: calling the API per request timed out the function and hit rate limits, so I split the paths — a daily cron pulls a rolling window, normalizes messy nested data, and writes one snapshot; the read endpoint serves that snapshot, fast and never rate-limited. OAuth 1.0 client with built-in rate limiting, 429 handling, and a bearer-auth-gated cron.

Third-party API integration Serverless + cron Read-path optimization
